# LoveOS 💕 - Native Android App for Couples

LoveOS — это нативное Android-приложение, созданное для укрепления отношений между парами через общие активности, отслеживание здоровья и бесшовное общение. Приложение построено на **Jetpack Compose** и использует современный дизайн "Premium Romantic" с элементами стеклянного морфизма (Glassmorphism) и микро-анимациями.

---

## 🏗️ Техническая Архитектура
Проект следует принципам **Clean Architecture** с четким разделением на слои: `data`, `domain` и `ui`.

### Структура пакетов
```text
com.example.loveosapk/
├── data/                        # Слой данных
│   ├── local/                   # Room DB (Сущности, DAO, База данных)
│   ├── remote/                  # Интеграция с Firebase Realtime Database
│   ├── repository/              # Реализации репозиториев (SSOT)
│   ├── Models.kt                # DTO и внутренние модели данных
│   └── PreferenceManager.kt     # Управление настройками через DataStore
├── domain/                      # Бизнес-логика (Pure Kotlin)
│   ├── model/                   # Доменные модели
│   ├── repository/              # Интерфейсы репозиториев
│   └── usecase/                 # Чистая логика (UseCases, например, расчет цикла)
└── ui/                          # Слой представления (UI)
    ├── components/              # Переиспользуемые компоненты и анимации
    ├── features/                # Функциональные фичи (Цикл и т.д.)
    ├── navigation/              # Навигация и графы экранов
    ├── screens/                 # Основные экраны приложения
    ├── theme/                   # Дизайн-система (Цвета, Типографика, Темы)
    └── MainViewModel.kt         # Центральная ViewModel приложения
```

## 🚀 Технологический стек
*   **Язык**: Kotlin
*   **UI**: Jetpack Compose (Material 3)
*   **Навигация**: Navigation Compose + Shared Element Transitions (Experimental)
*   **База данных**: Room (Локальное хранение)
*   **Облако/Синхронизация**: Firebase Realtime Database
*   **Инъекция зависимостей**: Ручная (через Factory) и Singleton паттерны
*   **Асинхронность**: Kotlin Coroutines & Flow
*   **Хранение настроек**: Jetpack DataStore
*   **Загрузка изображений**: Coil 3
*   **Мультимедиа**: Media3 (ExoPlayer)

## ✨ Основные функции

### 1. Главная (Home)
*   **Счетчик отношений**: Отображение времени вместе в днях, годах и месяцах.
*   **Таймер встречи**: Обратный отсчет до следующего свидания.
*   **Рулетка идей**: Генератор случайных занятий для пары.
*   **Сердечки**: Возможность отправлять анимированные "всплески" сердечек партнеру.

### 2. Женский календарь (Cycle)
*   Интерактивный календарь с фазами (менструация, овуляция).
*   Логирование симптомов и настроения.
*   **Partner Sync**: Партнер видит текущую фазу и состояние для проявления заботы.

### 3. Общие инструменты
*   **Список желаний (Wishlist)**: Совместное управление хотелками.
*   **Задачи**: Список дел с назначением исполнителя.
*   **Накопления**: Трекер финансовых целей.
*   **Заметки**: Общие записи и "Капсулы времени".

### 4. Развлечения (Games)
*   Крестики-нолики.
*   Виселица (Hangman) с подсказками.

## 🎨 Дизайн-система "Premium Romantic"
Приложение использует кастомную тему, настроенную в `ui/theme`:
*   **Акцентные цвета**: Romantic Pink (`#E91E63`) и Accent Purple.
*   **Glassmorphism**: Использование полупрозрачных поверхностей с размытием фона и тонкими границами.
*   **Микро-анимации**:
    *   `bounceClickable` — пружинящий эффект при нажатии.
    *   `OdometerCounter` — плавная смена цифр в счетчиках.
    *   `AtmosphericBackground` — анимированные частицы на фоне.

## 🛠️ Инструкция по доработке
Если вы хотите добавить новую функциональность:

1.  **Данные**: Добавьте новую сущность в `data/local/entities` и обновите `LoveOsDatabase`.
2.  **Бизнес-логика**: Создайте интерфейс в `domain/repository` и UseCase в `domain/usecase`.
3.  **UI**: 
    *   Создайте новый экран в `ui/screens`.
    *   Добавьте маршрут в `ui/navigation/Screen.kt`.
    *   Зарегистрируйте экран в `NavHost` внутри `MainActivity.kt`.
4.  **Синхронизация**: Для облачной синхронизации используйте `SyncRepository`, который связывает локальный Room и удаленный Firebase.

## ⚠️ Известные особенности
*   **Shared Transitions**: Используется экспериментальный API `SharedTransitionLayout`. При навигации будьте осторожны с именами ключей для `sharedElement`.
*   **Debug Mode**: В меню "Ещё" можно активировать режим разработчика, нажав на версию приложения 7 раз.
